Manía de apt-get

  • 2014-07-25
  • #Linux
  • #Ubuntu
  • #debian
Sólo una breve reflexión sobre los bloggers.

Ya han pasado tres meses desde el lanzamiento de Ubuntu 14.04 y sigo viendo en todos los blogs que todavía utilizan el comando apt-get para instalar programas desde los repositorios. Se me plantea la duda de si lo hacen porque no saben que la última versión de Ubuntu trae el nuevo comando apt o porque quieren asegurar la retrocompatibilidad de sus comandos con los lectores que utilicen versiones antiguas del sistema operativo.

Personalmente, llevo usando apt desde el primer día. Supongo que todos mis lectores son lo suficientemente inteligentes como para instalar la última versión de Ubuntu (Debian o Linux Mint) o que sepan detectar ese problema de incompatibilidad con apt-get. Tarde o temprano, todos los blogs tendrán que dar el paso y olvidarse de apt-get, como en su día nos olvidamos de aptitude, porque la diferencia entre apt y apt-get no está solo en la forma de invocarlos. El manual de apt lo explica perfectamente:
The apt command is meant to be pleasant for end users and does not need to be backward compatible like apt-get. Therefore some options are different: 
  • The option DPkgPM::Progress-Fancy is enabled.
  • The option APT::Color is enabled.
  • A new list command is available similar to dpkg --list.
  • The option upgrade has --with-new-pkgs enabled by default.
Me parecen suficientes razones como para relegar a apt-get al olvido, pero esto solo es una opinión personal.
