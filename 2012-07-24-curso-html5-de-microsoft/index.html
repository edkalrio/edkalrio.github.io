Curso HTML5 de Microsoft

  • 2012-07-24
    • Tweet
Como publicamos ayer por twitter, os traemos un tutorial, por parte de Microsoft, sobre HTML5, pero no sin antes hablar del asunto que me ha llevado a publicar esto.

Me parece bien, por parte de Microsoft, que cree tutoriales gratuitos para intentar enseñar a la gente determinadas cosas, como en este caso, a crear webs con HTML5, o como puede ser una resolución de un bug, o como instalar un Service Pack de eso que tanto les gusta. Entonces, ¿Cual es el problema?

Desde RW Labs estamos desarrollando una web desde cero sin mas ayuda que el software Blue Fish, que es poco mas que usar un bloc de notas, es decir, estamos creando una web en html5 y css3 con papel y lápiz. Todo funciona genial, poco a poco coge forma, hasta que por fin la tenemos terminada y subida ya al servidor. La web ha pasado el test de w3s y está perfectamente estructurada. La web funciona genial en Chrome, Firefox, navegadores móvil... pero ¿Que pasa con IE? No carga el CSS, por lo que la web se carga sin estilo, a lo bruto. 

¿Por que? Nadie lo sabe, una web realizada con todos los estándares y que pasa todos los tests de w3s sin un solo warning, y que no funcione en el navegador mas común entre todos los usuarios. Lo que una persona normal podría decir es que los usuarios de IE que les den y actualicen, pero esa opción no es viable para la empresa para la que estamos realizando la web, el empresario es quien debe adaptarse al cliente, no al revés, sino es muy probable que sea un cliente perdido.

Ahora estamos pensando que hacer con IE, si sacrificarlo a los dioses del metal, o por el contrario, crear un css especifico para IE, o forzar la instalación de chome frame para los usuarios de IE...

Y es que es curioso, que una compañía, cuyo software no es capaz de adaptarse a los estándares, te publique un tutorial sobre HTML5.

Aun así, lo prometido es deuda, aquí os dejo el tutorial, por si alguno de vosotros quiere introducirse en el mundo de HTML5.
